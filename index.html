Sistema de Agendamento de Compromissos

1. Descrição e Modelagem do Problema

Cenário Escolhido

O problema está relacionado a um sistema de agendamento de compromissos. Em um cenário típico, temos usuários que desejam agendar compromissos (como consultas, reuniões, entrevistas, etc.) com profissionais ou serviços, e o sistema precisa gerenciar a disponibilidade desses profissionais e os horários preferenciais dos usuários.

Entidades

  • Usuário: Pessoa que deseja agendar um compromisso.
  • Profissional: Pessoa ou entidade que oferece o serviço e tem um horário disponível para agendamento.
  • Compromisso: O evento que ocorre quando um usuário e um profissional se encontram para realizar um serviço.
  • Horário: Momento específico em que o compromisso está marcado.

Relações

  • Agendamento: Relaciona um usuário a um profissional em um horário específico.
  • Disponibilidade: Relaciona um profissional a um conjunto de horários disponíveis para compromissos.
  • Preferência de Horário: Relaciona um usuário a horários preferenciais, que podem ser um ou mais.

Restrições

  • Um usuário não pode agendar mais de um compromisso no mesmo horário.
  • Um profissional não pode atender a dois usuários no mesmo horário.
  • O sistema deve garantir que a disponibilidade do profissional seja respeitada.

Regras

  • Um usuário pode agendar um compromisso com um profissional, se o horário desejado pelo usuário estiver disponível.
  • Se o horário do profissional já estiver ocupado, o agendamento deve ser redirecionado para outro horário disponível ou o usuário deve ser notificado.

Versão Formal

Seja (U) um conjunto de usuários, (P) um conjunto de profissionais e (H) um conjunto de horários. A relação de agendamento (A(u, p, h)) indica que o usuário (u) agendou um compromisso com o profissional (p) no horário (h).

        (A(u, p, h)): Agendamento de (u) com (p) no horário (h).
        (D(p, h)): Disponibilidade de (p) no horário (h).
        (P(u, h)): Preferência de horário de (u) no horário (h).

O sistema deve garantir que (A(u, p, h)) só seja verdadeiro se (D(p, h)) for verdadeiro e o horário não conflitar com outros agendamentos.

2. Lógica Proposicional

Proposições

  • (P₁): O usuário (u) agendou um compromisso com o profissional (p) no horário (h).
  • (P₂): O horário (h) está disponível para o profissional (p).
  • (P₃): O usuário (u) prefere o horário (h).

Fórmulas Bem Formadas (FBFs)

  • (P₁ → P₂): Se o usuário agendou um compromisso, então o horário deve estar disponível.
  • (P₁ → ¬P₃): Se o usuário agendou, então o horário não pode ser o preferido.

3. Dedução Natural

Dedução 1

Se o usuário agendou um compromisso e o horário está disponível, então o horário deve estar disponível.

Dedução 2

Se o usuário agendou e o horário não é o preferido, então o usuário não prefere o horário.

4. Resolução Proposicional

Conversão para CNF

            (P₁ → P₂) → (¬P₁ ∨ P₂)
            (P₁ → ¬P₃) → (¬P₁ ∨ ¬P₃)

Resolução

Aplicando a resolução, conseguimos provar que o horário está disponível.

5. Lógica de Predicados

Definição de Predicados

  • Agendar(u, p, h): O usuário (u) agendou um compromisso com o profissional (p) no horário (h).
  • Disponivel(p, h): O profissional (p) está disponível no horário (h).
  • Preferencia(u, h): O usuário (u) prefere o horário (h).

Formalização em Primeira Ordem

            ∀u ∀p ∀h (Agendar(u, p, h) → Disponivel(p, h))
            ∀u ∃h (Preferencia(u, h))
